Set Up a Blog with Hexo!

十二月 26, 2021
  • -Techniques

To set up a blog powered by Hexo, you needed to do the things below:

  1. Download Node.js

  2. Download Git (if you haven’t downloaded it before)

  3. Install Hexo

  4. Set up a personal repository in GitHub

  5. Generate an SSH and add it to GitHub

  6. Deploy Hexo to GitHub

  7. Issue articles

  8. Other things you can do to make your blog more interesting

1. Download Node.js

Windows users:

You can get Node.js through the official website: https://nodejs.org/en/

Remember to download the LTS version. The LTS version is recommended because it has better compatibility.

Simply setting all the things as default values is OK.

If you want to install Node.js to a folder other than suggested, you should keep in mind that you should modify Node.js’s environment.

For Windows users, the following article is recommended to install Node.js: https://www.cnblogs.com/liuqiyun/p/8133904.html

Linux users:

sudo apt-get install nodejs

sudo apt-get install npm

The two commands can do everything for you.

Verification:

To verify you have successfully downloaded Node.js, you can open the terminal and text in

node -v

npm -v

If a version number is returned, you have successfully installed Node.js. Otherwise, for windows users, you can use the downloaded package to uninstall what you have installed and install again.

2. Download Git

Windows users:

You can get Git through the official website: https://gitforwindows.org/

Then do what the instructions ask you to do.

Linux users:

sudo apt-get install git

The command can do everything for you.

Verification:

git --version

Type the code into the terminal. If the version number of Git is returned, you have successfully installed Git.

3. Install Hexo

Warning: This part can be extremely troublesome!

If you choose to install Node.js in a folder suggested by the install package, congratulations! You have avoided a lot of trouble!

First, you should set up a place for your blog to be stored on your computer. You can create a folder somewhere on your computer.

Then, you should go to the folder in the terminal (using the command cd).

Next, type in the command npm install -g hexo-cli

Notice that -g is an alternative parameter, which means installing Hexo globally. And -cli allows you to use command lines to operate Hexo.

Verification:

Type in the code hexo -v in the terminal. If the version number and some other information are returned, you’ve successfully installed Hexo.

Default:

npm ERR! code EPERM
npm ERR! errno -4048

If these kinds of results occurred during your installation, you can find if there are some .npmrc documents generated. Remember, the extra .npmrc should not be what is in your “blog” folder. Instead, try to find these files in C:\Users\YourUserName.

If you do have solved the problems above but still can’t install, it might be that you’re not permitted. To solve it, you can open the WindowsPowerShell in your folder (remember to choose “Run as Administrator” ).

If the two ways cannot solve your problem, you may use the downloaded package to uninstall Node.js and install it again. This actually works even it sounds a bit irrelevant.

Initialize Hexo

Type in the code hexo init in the terminal can initialize your blog. Remember, you should be under the folder of your blog in the terminal when you commit to this command.

Wait for things being downloaded to your folder. If some errors occurred, check if you can connect GitHub. Simply typing in the initializing code again can be helpful.

periodical achievement

To see what you have done after so many steps, you can run hexo s to let your computer act as a server temporarily.

Type the address “localhost:4000” in a browser and you’ll see your initialized Hexo page.

4. Set up a personal repository in GitHub

Go to your own GitHub page (if you don’t have an account, just sign up for one and follow the instructions).

Choose the “Repositories” and click the new in green (in the right upper corner in this screenshot).

Remember to set the name of the repository as YourGitHubName.github.io. (For example, my GitHub account is Nephrn17, so the repository’s name is Nephren17.github.io) This can help you avoid later troubles.

5. Generate an SSH and add it to GitHub

After creating your own GitHub repository, you should link the repository with the blog on your computer. This is done by adding the SSH to GitHub. Open your blog’s folder, click in with the right button, then git bash here.

First, you should config your account. You can type the code in Git bash like this:

git config --global user.name "your GitHub user name"

git config --global user.email "your GitHub account's email"

After that, you can create your SSH.

ssh-keygen -t rsa -C "your GitHub account's email"

In fact, you can type nothing to create an SSH and that’s OK.

After that, a file is created to store the SSH. You can find it in C:\Users\YourUserName\.ssh. There’ll be a file named “id_rsa.pub”. Open the file and copy all the content. (Ctrl+A and Ctrl+C is recommended) Then paste the content to the GitHub page.

The “Title” is not important and you can choose anything you like to name it. Pate what you copied in the “Key”.

To verify if you succeeded, you can type ssh -T git@github.com to test.

6. Deploy Hexo to GitHub

Open the _config.yml file and change it a little bit. (You may need a VSCode to modify it. Some other alternatives are also OK.) You can find what is shown in the picture at the end of this file.

Notice that the “repository” should be your repository’s address (Not mine). Other things can

be simply copied.

Then, you need to install deploy-git to enable you to deploy what you have written on your computer to your GitHub

repository. Open the git bash in your blog’s file. The installation is quite simple by typing in a command:

npm install hexo-deployer-git --save

Then you can see what you have done by typing these commands:

hexo g

hexo d

If things go wrong in this step, still, it is probably caused by the network connection problems. You can now visit your website “www.YourGitHubUserName.github.io" and it is your website.

Note: for hexo g, the “g” stands for “generate”; for hexo d, the “d” stands for “deploy”. For more information about the command lines, you can use the code hexo -h or visit the official website: https://hexo.io/zh-cn/docs/commands

7. Issue Articles

To issue an article, you can write in the command line: hexo new "your article's name" and YourArticleName.md file will be generated in your “source” folder. You can change the file as you like to be the content of your blog. Then, you still need hexo g and hexo d to deploy it to your GitHub repository. If nothing is changed, you can simply do that again.

Some websites may suggest you use a hexo clean first. But this command may clear what you written before. So I suggest that if simply two commands are enough to reach your goal, there’s no need to add an extra one.

If you want to see what your article would be like before posting it, similarly, you can use hexo s and visit the local website. Remember, don’t use Ctrl+c to copy as this is meant to stop the program in the Git environment.

8. Other things you can do to make your blog more interesting

To make your blog more interesting, there’re many things that you can do. You can add a lot of plugins and themes to make your blog more attractive.

For plugins, you can visit the official website: https://hexo.io/plugins/ Actually, we have installed some plugins like hexo-cli and hexo-deployer-git. As all plugins process their own unique function, it’s a wiser choice to explore the world of plugins yourself.

For themes, you can find piles of them also on the official website and GitHub. The official website: https://hexo.io/themes/ The default theme is landscape. To change one, you need to download one first. You can go to chosen theme’s Github page and then clone its address.

Click the copy button like what’s on the right side of the picture. Then type git clone https://github.com/liuyib/hexo-theme-stun.git. (You can choose a different theme by doing what is similar).

The theme this blog is using is called diaspora, and you can find it easily on GitHub.
